# Active Directory Lab – Week 3: Group Policy Administration (GPO)

## Overview
This project builds on an Active Directory environment using a Domain Controller (DC01) and a domain-joined client machine (CLIENT01). The focus of Week 3 is Group Policy Object (GPO) administration, system control, and centralized configuration management.

---

## Environment
- DC01 (Domain Controller)
- CLIENT01 (Domain-joined workstation)
- Domain: corp.geekytech.local

---

## Skills Covered
- Group Policy Object (GPO) creation and management
- Password and account policy enforcement
- Desktop and system restriction configuration
- Security baseline enforcement
- OU-based policy targeting
- GPO troubleshooting and validation

---

## Week 3 Breakdown

### Day 1 – GPO Introduction and Setup
- Introduction to Group Policy Management Console (GPMC)
- Linking GPOs to domain and OUs
- Understanding inheritance and scope

### Day 2 – Password and Account Policies
- Password complexity rules
- Account lockout policies
- Security policy enforcement

### Day 3 – Desktop Restrictions
- Control Panel restrictions
- CMD and Run command restrictions
- Desktop environment lockdown

### Day 4 – Security and System Restrictions
- USB/device restrictions
- Software execution policies
- System-level controls

### Day 5 – OU-Based Policy Targeting
- Applying GPOs to specific OUs
- Department-based policy separation
- Group filtering and delegation concepts

### Day 6 – GPO Troubleshooting and Validation
- Using gpresult / rsop
- Troubleshooting policy application issues
- Verifying applied policies on client machines

---

## Outcome
By the end of Week 3, a fully functional Group Policy environment is configured, demonstrating centralized control over users and computers within an Active Directory domain.
